OpenICE User Manual

Introduction

Thanks for using OpenICE! This guide aims to introduce readers to medical device integration using the Open-source Integrated Clinical Environment (OpenICE). OpenICE is a distributed system platform that connects system nodes together - like medical devices found in an OR or ICU, applications for functions like clinical decision support (CDS), and external interfaces to other healthcare IT (HIT) systems like EMRs. OpenICE automates peer-to-peer node discovery, data publishing and subscribing, as well as proprietary medical device protocol translation. Users and developers of the OpenICE platform get the data they need, in a common sensible format, delivered to the destinations they want.

OpenICE has been called an open clinical research platform, an abstraction layer for medical devices, a medical Internet of things (MIoT) platform, a cyber-physical system (CPS) test tool, amongst other names. The current version of OpenICE enables users to convert heterogeneous medical device data into a common structure and protocol, and to exchange that data with clinical applications on a different machine (or machines). OpenICE includes several simple but useful applications as well as example code to help users to write their own.


System Overview

The OpenICE application is the entry and exit point for medical device data into and out of the network. OpenICE has two operating modes: Supervisor and Device-Adapter. The Device-Adapter software acts as a bridge that will to connect both real and simulated medical devices into the OpenICE network. The Device-Adapter software translates the proprietary device communication protocols into the standard OpenICE data structures (described in ice.idl) and communication protocol. The Supervisor hosts clinical applications and subscribes to all medical devices in the same Domain and Partition.

Running OpenICE from the GUI

Launch OpenICE using your preferred application start method. When opened, the OpenICE application presents a start menu GUI. This menu is used to configure the mode of operation of the OpenICE application.

Application
Choose which software mode to run the application as: ICE Supervisor or ICE Device Interface. The ICE_Supervisor hosts clinical applications. The ICE_Device_Interface connects a medical device to the OpenICE network.
Domain ID
OpenICE systems on the same network can coexist by using different integer domain identifiers. If there are no other ICE systems on your network then any value may be used. The default value used at MD PnP is 15.
FHIR Server
This is the base URL of an HL7 FHIR (Fast Healthcare Interoperability Resources) server from which patient information should be received.This field can be left blank to use a local database of patients.
NOTE: If you would like to use the GUI run both the Supervisor and a Device-Adapter(s), run the Supervisor first and start Device-Adapters from within the Supervisor.

OpenICE Supervisor

Devices

On the main screen of the ICE Supervisor, there will be a box on the right with the heading “Devices assigned to [insert name or unassigned]”. Text will appear in the center of the box if no devices are associated with the patient selected.

To connect a device, it must be assigned to a specific patient [to select a different patient see section II.C.1.]. After selecting a patient, then click on the “Create an ICE Device Adapter…” button at the bottom of the page. To read instructions on how to create an ICE Device Adapter see section II.C.2.

Once the ICE Device Adapter is created on the ICE Supervisor, an image of the device will appear in the devices box. Click on the image to make a new window appear that will give further device information and may also display waveforms (depending on the device). To disconnect the patient from the device, then select the large red button with an X.

Select a Patient

Devices must be assigned to a specific patient. To select a patient, click the drop down button and choose a patient. When the main screen of the ICE Supervisor is first opened, then the patient will be unassigned.

Create an ICE Device Adapter:

Click on the Create an ICE Device Adapter button to connect a patient to a device. This may be done on the main screen of the ICE Supervisor or while using an app. To do this, select a Device Category.

Serial:
Allows a device to the OpenICE apps through a serial connection.
Simulated:
Devices with simulated data may be used if no other devices are available. You may change the parameters on the simulated devices by using the simulation control app [see Section II.C.10].
Network:
Allows a device to the OpenICE apps through a network connection.

Once a device category is selected, then click the down arrow for next to Device Type to populate a list of devices. Select a device and then click the start button to run the ICE Device Adapter. Before running an adapter using a network or serial connection, the IP address must be included.

The new device should appear in the main screen of the ICE Supervisor and also in an app. Several devices may be connected to one patient. Devices cannot be disconnected while using an app. When the app is closed the new device will also be shown on the main screen of the ICE Supervisor.
